main.py → interviewer.py

Session bootstrap

main.py reads CLI flags, loads the JD and resume, chooses the backend, and constructs the interviewer object. This is the session entry point.

interviewer.py → context_parser.py

Interview context

The interviewer hands JD and resume text to context_parser.py so the system can infer competencies, tailor the question, and avoid generic prompts.

context_parser.py → llm.py

LLM-based parsing

The parser relies on LLMClient for structured extraction. That keeps the parsing logic reusable and isolated from the flow controller.

interviewer.py → prompts.py

Prompt management

prompts.py owns the system prompt, greeting, question, hint, and observation templates. This lets you tune interview behavior without editing state-machine logic.

interviewer.py → llm.py

Conversation engine

The interviewer delegates response generation and streaming to the model client. The flow decides when to speak; llm.py decides what to say.

interviewer.py → report_generator.py

Final evaluation

At the end of the session, the transcript and final code are pushed into report_generator.py to create a structured, explainable PDF.

agent.py / agent 2.py → openrouter_client.py

Panel branch

The LangGraph / LiveKit branch routes node-level decisions through openrouter_client.py. That is the natural home for your moderator, specialist agents, and routing logic.

agent.py / agent 2.py → storage.py

Session memory

Both agent files should write session events, weak areas, and scores to storage.py. That gives the system memory across turns and supports analytics.

LangGraph state machine

Planner Node

Builds the session plan

This node turns resume + JD into competencies and a plan of attack. It is the bridge between raw input and a role-specific interview flow.

Moderator Node

Chooses the next speaker

The moderator is the brain of the graph. It decides which interviewer should speak next and why, based on the current state and weak areas.

DSA / System / Behavioral Agents

Specialist questioners

Each specialist asks one focused question and keeps the conversation on its own domain. This is what turns the system into a true panel.

Evaluator Node

Scores the answer

The evaluator classifies the response depth, assigns a score, and captures evidence or gaps. That signal drives the next step.

Tracker Node

Controls the loop

The tracker decides whether to continue, switch topic, or end the interview. It is what makes the flow adaptive rather than linear.

End Session → Final Report

Explainable output

Once the tracker ends the session, the system generates the final report. This is where the interview becomes a product artifact instead of just a chat log.
